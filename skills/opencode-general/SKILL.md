# OpenCode General Skill

Use this for broad tasks without a clearer specialist.

## Workflow

1. Clarify success criteria.
2. Inspect only needed local files/docs.
3. Use external research when local knowledge is insufficient.
4. For UI/browser evidence, use stable wait-stabilize-scroll-settle capture.
5. Report findings, changes, commands, and risks concisely.

## Routing

UI/reference → designer. Implementation/tests → fixer/build. Architecture/review → oracle. Docs/library → librarian. Codebase search → explorer. High-stakes consensus → council.

Avoid loading many overlapping skills and ask when material ambiguity blocks quality.
